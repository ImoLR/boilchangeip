# Boil v2.1.1

## 修复

- 修复更新脚本依赖本地长期源码目录，导致普通安装用户无法更新的问题。
- `install.sh`、`update.sh` 和 `uninstall.sh` 现在均可从任意工作目录独立执行。
- 安装、更新和卸载流程不再依赖调用者当前目录。
- 统一脚本中的 GitHub 仓库、二进制路径、配置路径和 systemd 服务名称。
- 更新时继续保留 `/etc/boil/config.env`。
- 更新失败时恢复旧二进制并尝试重新启动服务。
- 更新失败时恢复配置备份，避免配置目录出现部分更新状态。

## 升级

现有用户可运行：

```bash
curl -fsSL https://raw.githubusercontent.com/ImoLR/boilchangeip/main/update.sh | sudo bash
```
