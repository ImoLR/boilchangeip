# Boil v2.1.2

## 修复

- 修复 `update.sh` 在 `sudo` 环境下找不到 `cargo`，导致普通用户无法升级的问题。
- `install.sh` 和 `update.sh` 现在默认优先下载 GitHub Release 预编译二进制。
- 普通用户执行一键安装或更新时，不再需要 Rust、Cargo、Git 或任何本地源码目录。
- 更新流程不依赖当前目录，也不依赖 `/opt/boilchangeip/source`。
- 源码编译只作为 Release 没有对应架构二进制时的兜底路径。
- 源码编译兜底时依次查找 `cargo`、`/root/.cargo/bin/cargo`、`/home/$SUDO_USER/.cargo/bin/cargo`。
- 更新时继续保留 `/etc/boil/config.env`。
- 更新失败时恢复旧二进制和配置备份，并尝试重新启动服务。

## Release Assets

本版本正式提供：

- `boil-linux-amd64`
- `boil-linux-arm64`

`install.sh` 和 `update.sh` 会自动识别系统架构并下载对应资产。

## 升级

现有用户可运行：

```bash
curl -fsSL https://raw.githubusercontent.com/ImoLR/boilchangeip/main/update.sh | sudo bash
```

## 安装

新用户可运行：

```bash
curl -fsSL https://raw.githubusercontent.com/ImoLR/boilchangeip/main/install.sh | sudo bash
```
