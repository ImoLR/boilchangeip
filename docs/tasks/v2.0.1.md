---

## 一、问题背景

目前执行：

```bash
boil change
```

有时会得到：

```text
success=0
unconfirmed=1
failed=0

post-change verification failed:
HTTP 400 Bad Request
内部错误，请稍后再试
```

但实际观察：

- change_ip() 已成功提交。
- 剩余次数已经减少。
- 稍后执行：

```bash
boil status
```

可以看到 IP 已经变化。

因此：

真正失败的是立即验证。

不是换 IP 本身失败。

---

## 二、修复目标

必须保证：

- change_ip() 每次操作只能调用一次。
- 禁止自动重试 change_ip()。
- 只允许重试 post-change 的 get_ip()。
- CLI、Telegram、Timer 使用统一逻辑。
- 不修改 Token API。
- 不修改配置格式。
- 不重构无关代码。

---

## 三、建议验证流程

```
读取旧 IP
    ↓
change_ip()（仅调用一次）
    ↓
等待约 2 秒
    ↓
get_ip()

如果：

- HTTP 400
- 临时 API 错误
- 返回旧 IP

则等待约 2 秒继续验证。

最多验证 3 次。
```

不得再次调用 change_ip()。

---

## 四、结果判定

### 成功

任意一次 get_ip() 返回新 IP：

- success=1
- 显示旧 IP
- 显示新 IP

---

### 已接受但未确认

连续最多 3 次仍无法确认：

- ChangeAcceptedButUnconfirmed
- unconfirmed=1
- 不得归类为 Failed

普通用户提示：

> 换 IP 请求已被接受，Boil 后端仍在切换，请稍后使用 `boil status` 或 Telegram `/status` 查看。

不要向普通用户直接显示完整 HTTP400 异常。

详细错误可以保留在 debug 日志。

---

### 真正失败

只有 change_ip() 本身失败时：

- Failed

不要把验证失败误判为换 IP 失败。
